# Электронный журнал (Web-based Gradebook)

Электронный журнал с двумя ролями — **Студент** и **Преподаватель**. Расписание,
журнал оценок и посещаемости, программа по предметам, лабораторные работы с
прикреплением файлов, командные работы, проверка и комментирование работ
преподавателем, а также графики успеваемости студента и группы.

Бело-красная адаптивная тема, всплывающие окна, подсветка строки/столбца/ячейки
в журнале, управление мышью (ЛКМ/СКМ/ПКМ) и клавиатурой.

## Технологии

| Слой        | Стек                                                            |
|-------------|-----------------------------------------------------------------|
| Фронтенд    | React 18 + Vite + TypeScript + Tailwind CSS, recharts, lucide   |
| Бэкенд      | Node.js + TypeScript + Express, zod (валидация)                 |
| База данных | PostgreSQL 16 (через Docker Compose)                            |
| Auth        | JWT в httpOnly cookie, разграничение ролей middleware           |
| Файлы       | Локальное хранилище (`/uploads`) через multer                   |

## Запуск

### Требования
- Node.js 18+
- Docker + Docker Compose

### 1. База данных

**Вариант A — Docker (как в плане):** из корня репозитория
```bash
docker compose up -d
```
Поднимется PostgreSQL на `localhost:5432` (БД/пользователь/пароль — `gradebook`).

**Вариант B — нативный PostgreSQL (без Docker):** установите PostgreSQL
(`winget install -e --id PostgreSQL.PostgreSQL.16`), затем создайте роль и базу
под суперпользователем `postgres`:
```sql
CREATE ROLE gradebook LOGIN PASSWORD 'gradebook';
CREATE DATABASE gradebook OWNER gradebook;
```
`server/.env` уже настроен на эти реквизиты (`localhost:5432`). Если порт/пароль
другие — поправьте `DATABASE_URL`.

### 2. Бэкенд
```bash
cd server
cp .env.example .env        # при необходимости отредактируйте
npm install
npm run migrate             # создаёт таблицы (schema.sql)
npm run seed                # демо-аккаунты и базовая структура
npm run dev                 # API на http://localhost:4000
```

### 3. Фронтенд
```bash
cd client
npm install
npm run dev                 # UI на http://localhost:5173
```
Vite проксирует `/api` на бэкенд, поэтому отдельная настройка CORS для разработки не нужна.

Откройте **http://localhost:5173**.

## Возможности по ролям

### Администратор
Управление справочниками через интерфейс (без ручного SQL):
- **Пользователи** — создание/редактирование/удаление преподавателей, студентов
  и админов; назначение группы и статуса (активный/новый/отчислен), смена пароля.
- **Группы** и **Предметы** — справочники.
- **Курсы** — связка «предмет + группа + преподаватель».
- **Расписание** — еженедельные занятия (видны студентам и преподавателям).

### Студент
- **Расписание** — актуальное недельное расписание.
- **Журнал** — таблица «предмет × дата» с оценками и пропусками.
- **Страница предмета** — успеваемость по типам работ + график динамики оценок.
- **Лабораторная** — прикрепление решения (файл/ссылка), теория и ТЗ,
  дата выдачи, дедлайн, напарники (для командных работ), отметка и комментарий
  преподавателя.

### Преподаватель
- **Расписание** — занятия преподавателя.
- **Журнал** (предметы и группы):
  - добавление дня (урока);
  - **ЛКМ** — оценка (всплывающее окно или ввод с клавиатуры, только числа),
    **средняя кнопка** — опоздание, **правая кнопка** — отсутствие;
  - кнопка «Авто» — определение опоздания по таймингам урока (`bell_schedule`);
  - обесцвечивание отчисленных и выделение новых студентов;
  - подсветка ячейки, строки и столбца при наведении.
- **Программа по предмету** — шаблон занятий (лабы/теория/практика/контрольные/
  тесты), дедлайны, комментарии, загрузка ТЗ, настройка команд.
- **Сдача работ** — список сдавших, сроки (с пометкой просрочки), ссылки на
  файлы/решения, проверка с выставлением отметки и комментария.
- **Графики** — средний балл по студентам, динамика и распределение оценок группы.

### Общие возможности (все роли)
- 🔔 **Уведомления** — колокольчик в шапке со счётчиком непрочитанных: новая
  оценка, проверена работа, приближается дедлайн (генерируется автоматически для
  работ со сроком в ближайшие 3 дня). Поллинг каждые 30 c, «прочитать все».
- 📊 **Экспорт в Excel** — журнал преподавателя, ведомость сдач, личные оценки
  студента (`.xlsx`, ExcelJS). Кнопка **PDF** — печать/сохранение через браузер.
- 👤 **Личный кабинет** (`/profile`) — профиль и смена собственного пароля.

**Безопасность:** `helmet`, ограничение попыток входа (`express-rate-limit`,
10 запросов / 15 мин), валидация типов загружаемых файлов (белый список
расширений) и лимит размера.

## REST API (кратко)

| Метод | Маршрут | Назначение |
|-------|---------|------------|
| POST | `/api/auth/login` `/logout` · GET `/me` | аутентификация |
| GET  | `/api/schedule` | расписание текущего пользователя |
| GET  | `/api/student/journal` · `/courses/:id` | журнал и успеваемость студента |
| GET  | `/api/courses` · `/:id/journal` · `/:id/students` | курсы и журнал преподавателя |
| POST/DELETE | `/api/courses/:id/lessons` | управление днями журнала |
| PUT  | `/api/grades` | выставление оценки/посещаемости |
| CRUD | `/api/assignments` (+ `/:id/files`, `/:id/teams`) | программа и команды |
| GET/POST | `/api/submissions` · PUT `/:id/review` | сдачи и проверка |
| POST/GET | `/api/files` · `/:id` | загрузка и скачивание файлов |
| GET  | `/api/analytics/student/:id` · `/group/:id` | данные для графиков |
| CRUD | `/api/admin/users` · `/groups` · `/subjects` · `/courses` · `/timetable` | админ-панель (роль admin) |
| GET/POST | `/api/notifications` · `/read` · `/:id/read` | уведомления |
| GET | `/api/export/journal/:id` · `/submissions/:id` · `/student-journal` | выгрузка в Excel |
| POST | `/api/auth/password` | смена своего пароля |

Все защищённые маршруты требуют валидной cookie-сессии; преподавательские —
проверяют владение курсом, студенческие — принадлежность к группе. Тела запросов
валидируются на сервере (zod) и на клиенте.

## Сборка production

```bash
cd client && npm run build                # сборка статики в client/dist
cd server && npm run build && npm start   # tsc → dist (+ schema.sql), запуск
```
В продакшене Express сам отдаёт собранный фронт (`CLIENT_DIST`) одним origin'ом —
отдельный веб-сервер для статики не нужен.

## Тайминги уроков (авто-опоздания)

Расписание звонков хранится в таблице `bell_schedule` и заполняется сидом
(6 пар). Кнопка «Авто» в ячейке журнала сравнивает текущее время с временем
начала/конца урока и проставляет «присутствовал / опоздание / отсутствие».
Порог опоздания — `LATE_THRESHOLD_MIN` в `server/src/utils/timing.ts`.
